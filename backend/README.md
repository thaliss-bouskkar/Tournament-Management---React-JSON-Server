# Tournament Management Backend

This is the Node.js + Express + MongoDB backend that replaces `json-server`. 
It provides the exact same endpoints to maintain compatibility with your React frontend.

## Getting Started

1. **Install dependencies:**
   ```bash
   cd backend
   npm install
   ```

2. **Database configuration:**
   Copy `.env.example` to `.env` and fill in your MongoDB connection string (`MONGO_URI`).
   For local development:
   ```env
   PORT=5000
   MONGO_URI=mongodb://127.0.0.1:27017/tournamentDB
   ```

3. **Seed your Database (Optional but recommended):**
   If you have a `db.json` file in the root of your project, you can automatically import it into MongoDB to keep all your existing data. Make sure your `.env` has the correct `MONGO_URI`, then run:
   ```bash
   npm run seed
   ```

4. **Running the server:**
   ```bash
   node server.js
   ```
   The backend will run on `http://localhost:5000`.

## Updating Your React App

To connect your existing React application to this new backend:

1. **Update API URLs**: Find where your React app makes API calls (e.g., using `fetch` or `axios` with `http://localhost:3000`). Change the base URL from `http://localhost:3000` to `http://localhost:5000` (or whatever URL your backend is hosted on).

Example (Using axios instances):
```javascript
const api = axios.create({
  baseURL: 'http://localhost:5000'
});
```

2. **Restart React App**: The endpoints provided (`/admins`, `/players`, `/teams`, `/groups`, `/matches`, `/statistics`) perfectly match the paths generated by `json-server`. 

## Deployment Instructions

### Deploying the Backend on Render
1. Create a [Render](https://render.com) account.
2. Link your GitHub repository.
3. Create a **New Web Service**.
4. Set the Root Directory to `backend` (if you push it as a subfolder).
5. Set Build Command: `npm install`
6. Set Start Command: `node server.js`
7. In the **Environment Variables** section, add your production MongoDB URI as `MONGO_URI`.

### Deploying the Frontend on Vercel
1. Create a [Vercel](https://vercel.com) account and link your GitHub.
2. Import the React project (point it to your root project folder).
3. Under **Environment Variables**, add the URL of your deployed Render backend (e.g., `VITE_API_URL=https://your-backend.onrender.com` or `REACT_APP_API_URL=...` depending on what build tool you use).
4. Update your React components to use this environment variable instead of a hardcoded URL.
5. Deploy the application.
